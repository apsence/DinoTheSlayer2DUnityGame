```markdown
# Основные команды Markdown для составления TODO-списков

## Чеклисты (стандарт GitHub Flavored Markdown)

В стандартном Markdown работают **только** два варианта:

```markdown
- [ ] Невыполненная задача
- [x] Выполненная задача
```

> ⚠️ **Важно:** Символы вроде `[!]`, `[?]`, `[~]` внутри чекбоксов **не работают** в обычном Markdown.  
> Для выделения важных задач используйте **жирный шрифт** или **Admonition-блоки** (см. раздел ниже).

## Admonition-блоки (цветные вставки)

Если нужно **выделить задачу, заметку или предупреждение** — используйте специальный синтаксис с `>` и квадратными скобками.

```markdown
> [!NOTE]
> Обычная заметка — синий блок.

> [!TIP]
> Полезный совет — зелёный блок.

> [!IMPORTANT]
> Критически важная информация — фиолетовый/синий блок.

> [!WARNING]
> Будьте осторожны — жёлтый/оранжевый блок.

> [!CAUTION]
> Опасность! — красный блок.
```

**Как это выглядит в деле (пример для TODO):**

```markdown
> [!IMPORTANT]
> **Срочно:** доделать префабы до вечернего билда!

> [!WARNING]
> Баг с синхронизацией тени может привести к вылету на Android.
```

## Заголовки (структурирование)

```markdown
# TODO List - Крупный раздел
## Проект - Подраздел
### Этап 1 - Детали
#### Конкретная задача
```

## Пример полного TODO-файла (с рабочим синтаксисом)

```markdown
# 📋 TODO Лист

## 🎯 Приоритетные задачи (сегодня)
- [ ] **Исправить баг с анимацией атаки**  ← жирное выделение вместо [!]
- [ ] **Настроить синхронизацию тени**
- [ ] Оптимизировать спавнер врагов

## 🚧 В процессе
- [ ] Рефакторинг системы здоровья (50% готово)
- [ ] Переделать систему диалогов (нужно уточнить ТЗ)

> [!NOTE]
> Для уточнения ТЗ по диалогам — созвон в 15:00.

## ✅ Завершено
- [x] Настроить передвижение игрока
- [x] Добавить спринт с кулдауном
- [x] Реализовать атаку

## 🐛 Баги
- [ ] Анимация даша не синхронизируется
- [ ] При атаке можно двигаться
- [ ] Странное поведение камеры (нужно воспроизвести)

> [!CAUTION]
> Критический баг: при переключении уровней теряется прогресс.

## 💡 Идеи
- [ ] Добавить систему сохранений
- [ ] Сделать разные типы атак
- [ ] Реализовать инвентарь
```

## Дополнительные элементы оформления

### Горизонтальные линии (разделители)
```markdown
---
```

### Выделение важного (рабочие варианты)
```markdown
**Жирный текст** - важная задача
*Курсив* - второстепенная
***Жирный курсив*** - очень важно
~~Зачёркнуто~~ - отменённая задача
`код` - техническая задача
```

### Списки
```markdown
1. Первый шаг
2. Второй шаг
3. Третий шаг

- Пункт маркированного списка
  - Вложенный пункт
    - Ещё глубже
```

### Цитаты для заметок (обычные, без иконок)
```markdown
> Примечание: не забыть протестировать на мобильном устройстве
> 
> Важно: обновить документацию после завершения
```

### Таблицы (для отслеживания прогресса)
```markdown
| Задача | Приоритет | Статус | Дата |
|--------|-----------|--------|------|
| Анимация атаки | Высокий | ✅ | 2024-01-15 |
| Система звука | Средний | 🔄 | 2024-01-16 |
| UI меню | Низкий | ⏳ | - |
```

### Эмодзи для визуального оформления
```markdown
🎯 Приоритетные задачи
🔄 В процессе
✅ Выполненные
🐛 Баги
💡 Идеи
📝 Заметки

❓ Требуют уточнения
🔨 Технические задачи
🎮 Игровой контент
🎨 Графика/Анимация
🔊 Звук
⚡ Оптимизация
📱 Платформа
🧪 Тестирование
```

## Полезные сервисы для работы с .md TODO:

```markdown

- [ ] #43 - При рывке можно атаковать

- [ ] Оптимизировать спавн объектов
